# 账号分配

> **前置条件：** 使用 `lark-cli` 从飞书多维表格获取账号数据，从 `C:\账号梳理\syetemid.txt` 查找 systemid。

## 数据源

飞书多维表格：**账号管理系统**
- Token: `AdSxbgrTgaVZY7sGGJicufPxn2F`
- 链接: https://dcndc1g978pn.feishu.cn/base/AdSxbgrTgaVZY7sGGJicufPxn2F

| 表名 | 用途 | 记录数 |
|------|------|--------|
| 顶点两融账号表 | 顶点两融账号 | 16 |
| 顶点现货账号表 | 顶点现货账号 | 6 |
| 低延时账号表 | 低延时账号(9691/96) | 974 |

## 表字段

### 顶点两融账号表 / 顶点现货账号表
- 客户号、资金账号、营业部、密码、上海证券股东账号/账户、深圳证券股东账号/账户、SystemID、是否已分配

### 低延时账号表
- 客户号、资金账号、营业部、密码、深圳证券股东账号、上海证券股东账号、systemid、是否已分配

## 分配状态

每个表都有「是否已分配」字段，用于跟踪账号分配状态：
- **未分配**：可分配给客户
- **已分配**：已分配给客户，不再返回

## 查询命令

```bash
# 列出表字段
lark-cli base +field-list --base-token AdSxbgrTgaVZY7sGGJicufPxn2F --table-id <表名>

# 查询所有记录
lark-cli base +record-list --base-token AdSxbgrTgaVZY7sGGJicufPxn2F --table-id <表名> --limit 500

# 按关键词搜索记录
lark-cli base +record-search --base-token AdSxbgrTgaVZY7sGGJicufPxn2F --table-id <表名> --keyword <关键词>
```

## 触发关键词与命令

### 自动分配（无需指定客户号）
| 命令 | 说明 |
|------|------|
| `分配一个顶点两融账号` | 自动分配一个未分配的顶点两融账号 |
| `分配一个顶点现货账号` | 自动分配一个未分配的顶点现货账号 |
| `分配一个低延时账号` | 自动分配一个未分配的低延时账号 |

### 按客户号查询
| 命令 | 说明 |
|------|------|
| `账号分配 顶点 两融 <客户号>` | 查询并分配顶点两融账号（自动跳过已分配） |
| `账号分配 顶点 现货 <客户号>` | 查询并分配顶点现货账号（自动跳过已分配） |
| `账号分配 低延时 客户号 <客户号>` | 查询低延时账号通过客户号（自动跳过已分配） |
| `账号分配 低延时 资金账号 <资金账号>` | 查询低延时账号通过资金账号（自动跳过已分配） |

**注意**：搜索时已分配的账号会被自动跳过，不会返回。分配后自动标记为「已分配」。

## 分配后操作

分配账号后，AI 需要在多维表格中将对应记录的「是否已分配」字段更新为「已分配」：

```bash
lark-cli base +record-upsert --base-token AdSxbgrTgaVZY7sGGJicufPxn2F --table-id <表名> --record-id <记录ID> --json "{\"是否已分配\":\"已分配\"}"
```
